# Security Policy

## Supported Versions

Only the latest version of Lucid-Lyrics is currently supported with security
updates. Because this is a customizable extension for Spotify, we highly
recommend always updating to the most recent release to ensure compatibility and
security.

| Version              | Supported          |
| -------------------- | ------------------ |
| Latest (main branch) | :white_check_mark: |
| Older releases       | :x:                |

## Reporting a Vulnerability

We take the security and privacy of Lucid-Lyrics users seriously. If you
discover a security vulnerability, please **do not open a public issue** as this
could expose users to the exploit before a patch is available.

Instead, please report the vulnerability using one of the following methods:

1. **GitHub Security Advisories (Preferred):** Navigate to the
   [Security tab](https://github.com/sanoojes/Lucid-Lyrics/security) of the
   Lucid-Lyrics repository and click on "Report a vulnerability" to privately
   disclose the issue directly to the maintainers.
2. **Direct Contact:** You can reach out to the developer directly via the
   contact methods listed on the
   [socials.sanooj.uk](https://socials.sanooj.uk) or through their provided
   Discord/social links.

**What to expect after reporting:**

- **Acknowledgement:** You can expect an initial reply acknowledging your report
  within 48-72 hours.
- **Triage & Updates:** We will investigate the issue and provide you with an
  update on whether the vulnerability is accepted, along with an estimated
  timeline for a fix.
- **Resolution:** If the vulnerability is confirmed, we will develop a patch,
  push a security update to the latest version, and publicly credit you in the
  release notes once the patch is safely deployed.
